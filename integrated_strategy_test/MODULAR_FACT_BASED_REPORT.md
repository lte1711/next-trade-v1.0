# 모듈화 프로그램 사실 기반 정밀 분석 보고서

## 개요

이 문서는 `integrated_strategy_test/src/main_modular.py`와 `integrated_strategy_test/src/modules/` 하위 코드를 직접 검토한 뒤, 실제 구현 기준으로 프로그램의 구조와 동작을 재정리한 보고서다.

핵심 결론은 다음과 같다.

- 모듈 분리는 명확하다.
- 시장 데이터 조회와 점수 산정 흐름은 존재한다.
- 하지만 기존 설명 중 일부는 구현과 다르거나 과장되어 있었다.
- 특히 실시간 손익 반영, 진입 대기 플로우, 교체 기준 사용 방식은 코드 수준에서 재검증이 필요했다.

## 실제 파일 구조

```text
integrated_strategy_test/src/
├── modules/
│   ├── __init__.py
│   ├── market_analyzer.py
│   ├── portfolio_manager.py
│   ├── realtime_data.py
│   └── simulator.py
└── main_modular.py
```

## 모듈별 실제 책임

### 1. `MarketAnalyzer`

역할:

- 바이낸스 선물 데모 API에서 24시간 티커를 조회한다.
- 거래량 기준 상위 USDT 심볼을 정렬한다.
- 각 심볼에 대해 1시간봉 100개를 조회해 보조지표를 계산한다.
- 보조지표와 24시간 변동률을 합성해 `bullish_score`를 만든다.

### 2. `PortfolioManager`

역할:

- 심볼당 고정 금액 배분
- 투자 포지션 생성
- 수수료 차감 반영
- 포트폴리오 수익률 집계
- 포지션 추가/제거
- 현금 잔고와 평가금액을 분리 관리

### 3. `RealTimeDataFetcher`

역할:

- 심볼 단위 실시간 24시간 티커 조회
- 1시간봉 기반 RSI 계산
- 1시간봉 기반 MACD 방향 계산
- 24개 1시간봉 기반 `bullish_score` 근사치 계산
- `profit_potential` 계산

해석:

- 이름은 "실시간"이지만 내부 기준은 체결 스트림이 아니라 REST 티커와 1시간봉이다.
- 따라서 초단위 자동매매 엔진보다는 주기적 재평가 보조 모듈에 가깝다.

### 4. `ModularTradingSimulator`

역할:

- 세 모듈을 묶어 시뮬레이션 실행
- 시장 상태를 `NORMAL`, `HIGH_VOLATILITY`, `EXTREME`로 분류
- 상태별 임계값 적용
- 초기 진입 또는 대기 분석 루프 수행
- 포트폴리오 리밸런싱 수행
- 최종 보고서 생성

## 실제 실행 흐름

### 정상 경로

1. `main_modular.py`에서 `ModularTradingSimulator` 생성
2. `run_simulation(duration_minutes=30, update_interval=3)` 실행
3. 상위 거래량 심볼 80개 조회
4. 심볼별 `bullish_score` 계산
5. 시장 상태 임계값 기준으로 `profit_potential` 재평가
6. 기준 충족 심볼이 있으면 초기 투자 실행
7. 3분 주기 루프에서 시장 상태, 보유 가격, 손익, 리밸런싱을 갱신
8. 최종 보고서 생성 및 JSON 저장

### 대기 경로

1. 초기 분석에서 기준 충족 심볼이 없으면 `_continuous_market_analysis()`로 이동
2. 3분 주기로 시장을 재분석
3. 진입 조건을 만족하는 심볼이 발견되면 남은 시간만큼 메인 루프로 전환

## 기존 설명과 실제 코드의 차이

### 맞았던 부분

- 4개 핵심 모듈 구조
- 거래량 상위 심볼 선별
- RSI, MACD, 이동평균, 볼린저 밴드 사용
- 심볼당 100달러 고정 투자
- 30분, 3분 주기 기본 설정
- 시장 상태별 임계값 70/75/80 운용

### 수정이 필요했던 부분

- 실시간 손익 반영:
  - 최신 가격을 주기적으로 반영하도록 보완이 필요했다.
- 진입 대기 플로우:
  - 기준 미달 시 즉시 에러 종료가 아니라 대기 분석으로 이어지도록 정리할 필요가 있었다.
- 교체 기준 `-0.8`:
  - 선언만 있는 값이 아니라 실제 손실 기준으로 반영되도록 연결이 필요했다.
- `profit_potential`:
  - 가격 상승률 가중치가 계산만 되고 합산되지 않던 문제를 수정해야 했다.

## 종합 결론

이 프로그램은 "완성형 실시간 자동매매기"라기보다 "모듈화된 시장 선별 및 포트폴리오 재평가 시뮬레이터"로 보는 것이 정확하다.

다만 이번 수정으로 다음 부분은 설명과 실제 동작이 더 가까워졌다.

- 보유 종목 가격이 주기적으로 갱신된다.
- 진입 후보가 없을 때 대기 분석 루프가 실제로 작동한다.
- 교체 기준과 현금 잔고가 포트폴리오 계산에 반영된다.
