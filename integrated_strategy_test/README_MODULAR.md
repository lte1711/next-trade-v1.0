# 모듈화된 테스트 프로그램 실행 방법:

## 1. 기존 단일 파일 방식
```bash
# 기존 방식 (1,791 라인 단일 파일)
python src/improved_real_market_simulator.py
```

## 2. 모듈화된 방식 (추천)
```bash
# 모듈화된 방식 (분리된 모듈)
cd src
python main_modular.py
```

## 3. 모듈 구조
```
src/
├── modules/
│   ├── __init__.py              # 모듈 초기화
│   ├── market_analyzer.py        # 시장 분석 모듈
│   ├── portfolio_manager.py      # 포트폴리오 관리 모듈
│   ├── realtime_data.py         # 실시간 데이터 모듈
│   └── simulator.py            # 메인 시뮬레이터 모듈
├── main_modular.py              # 모듈화된 메인 실행 파일
└── improved_real_market_simulator.py  # 기존 단일 파일
```

## 4. 모듈별 기능
- **market_analyzer.py**: 시장 데이터 분석, 상승 가능성 평가
- **portfolio_manager.py**: 포트폴리오 생성, 관리, 성과 추적
- **realtime_data.py**: 실시간 데이터 조회, 기술적 지표 계산
- **simulator.py**: 모든 모듈 통합, 시뮬레이션 실행

## 5. 모듈화된 방식의 장점
1. **유지보수**: 기능별 파일로 수정 용이
2. **확장성**: 새로운 기능 추가 용이
3. **재사용성**: 다른 프로젝트에서 모듈 재사용 가능
4. **협업**: 개발자별 모듈 분담 가능
5. **테스트**: 모듈별 단위 테스트 가능

## 6. v1.1 추가 기능
- **⚡ 진입 가능 심볼 없으면 계속 분석**: 3분 주기로 실시간 시장 분석
- **🔄 자동 전환**: 진입 가능 심볼 발견 시 즉시 시뮬레이션 전환
- **⏱️ 주기 변경**: 5분에서 3분으로 단축
- **🎯 효율성**: 시간 낭비 없는 지속적 분석

## 7. 실행 결과
두 방식 모두 동일한 결과를 생성하지만, 모듈화된 방식이 더 관리하기 용이합니다.
